# Commit and Push

1. Stage all changes: `git add -A`
2. Create commit with message from user input or generate descriptive message
3. Push to current branch: `git push`
4. Show result: `git log --oneline -1`

If user provides a message, use it. Otherwise, analyze the staged changes and create a descriptive commit message following conventional commits format.

Example: `feat(cmo): add sidebar component with 5 pillars structure`
